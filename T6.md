# T6 — Isomorfismo em Árvores

## Objetivo

Implementar um programa em Java que leia duas árvores não direcionadas no formato `algs4` e determine se elas são isomorfas usando codificação canônica.

## Conceito principal

Duas árvores são isomorfas quando possuem a mesma estrutura de conexões, mesmo que os nomes ou números dos vértices sejam diferentes.

Por isso, não basta olhar para o desenho, comparar apenas os graus ou usar uma busca simples. É necessário gerar uma representação estrutural independente da rotulação original dos vértices.

## Algoritmo usado

A solução segue estes passos:

1. ler os dois grafos usando `Graph.java`;
2. verificar se cada grafo é uma árvore válida;
3. encontrar o centro ou os dois centros de cada árvore;
4. codificar cada árvore a partir do centro;
5. ordenar os códigos dos filhos em cada chamada recursiva;
6. comparar as codificações canônicas finais.

## Validação da árvore

Uma entrada é considerada árvore quando:

- possui pelo menos um vértice;
- possui exatamente `V - 1` arestas;
- é conexa.

Se alguma entrada falhar nessa validação, o programa interrompe a comparação e informa o motivo.

## Centro da árvore

O centro é encontrado pela remoção iterativa das folhas. A cada rodada, os vértices de grau `0` ou `1` são removidos conceitualmente. No fim, sobra um centro ou dois centros.

## Codificação canônica

A codificação é feita de forma recursiva:

- uma folha recebe o código `()`;
- cada vértice codifica seus filhos;
- os códigos dos filhos são ordenados lexicograficamente;
- o resultado é colocado entre parênteses.

Essa ordenação é essencial porque a árvore pode ser lida em ordens diferentes no arquivo, mas a estrutura deve gerar sempre a mesma codificação.

## Exemplo de execução

```bash
javac src/*.java
java -cp src Main dados/tree3.txt dados/tree4.txt
```

Para as árvores `tree3.txt` e `tree4.txt`, o resultado esperado é que elas sejam isomorfas.
